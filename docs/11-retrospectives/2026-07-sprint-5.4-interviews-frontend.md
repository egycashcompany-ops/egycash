## 1. Sprint goal

Deliver the **Interviews** feature screens — the third Recruitment feature set — on the Phase 1
foundation, reusing the Applicants/Screening building blocks. **Interviews only**; no later stage.
Two authentication dev-login fixes surfaced during review and were folded into the same release.

## 2. Delivered features

- **Queue** (`interview.view`) — sortable `DataTable` (stage order, scheduled, created — the
  backend's sortable fields); filters (status + outcome + stage + an **applicant search-picker** →
  `applicantId` + a scheduled-date range); pagination — all **URL-synchronized**. **Schedule
  interview** dialog (applicant + stage + date/time + panel).
- **Detail** — the **interviewer panel with per-interviewer evaluation state**
  (recommend/neutral/notRecommend + rating + notes), the scheduling read-out, and the full action
  surface: **reschedule**, **reassign panel**, **skip** a pending interviewer (`interview.edit`),
  **submit/update your own evaluation** (`interview.evaluate`, assigned members only), **cancel**
  (`interview.cancel`), and **Pass / Fail** (`interview.decide`) — blocked with an inline notice
  while any panelist is still `pending`. Version-checked mutations.
- **Interviewer references** — a `UserPicker` / `UserName` pair reuses the platform Users endpoint
  (`user.view`) rather than exposing raw ids; degrades to a short reference without directory access.
- Feature `api/` layer + TanStack Query hooks against `/hr/interviews` (+ `/hr/interview-stages` and
  the action sub-routes); `ar` + `en` i18n; permission-gated throughout. **No new backend API.**
- **Auth — QR TOTP enrollment** (#38): the login 2FA enrollment step renders the backend
  `otpauthUrl` as a scannable QR (Microsoft Authenticator / any TOTP app), manual key kept as a
  fallback. One small runtime dep (`qrcode.react`, inline SVG).
- **Auth — dev-login fix** (#39): the seed now disables `auth.totp.enforcedForPrivileged` at
  organization scope so seeded (privileged) accounts can log in with email/password; production
  keeps the default. Seed data extracted to an importable `seed-data.ts` with a login regression
  test.

## 7. Lessons learned

1. **Reuse compounded again.** The applicant search-picker, URL-as-state list pattern, dialogs,
   `DataTable`, filters, and form primitives all came straight from Phases 1–3 — Interviews was
   mostly wiring plus the panel/evaluation/decision specifics.
2. **Evidence over assumptions.** The "login fails after seed" report was diagnosed by running the
   real `authService.login` and the settings registry — proving the exact branch (privileged +
   TOTP-enforced) rather than guessing, and the fix shipped with a regression test that reproduces
   the seed path so it can't silently regress.
3. **TOTP is clock-bound.** A real authenticator can't be used against a non-real-time dev clock;
   disabling enforcement in the dev seed is the honest fix, while the QR enrollment improves the
   real-user path where 2FA is genuinely enabled.
